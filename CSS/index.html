CSS Specificity

CSS Specificity - Theory

CSS specificity determines which styles are applied to an element when there are conflicting styles.

It is calculated based on the types of selectors used in a rule:

  • Inline styles have the highest specificity.
  • ID selectors have higher specificity than class selectors.
  • Class selectors have higher specificity than type selectors.

For example, if an element has both a class and an ID selector applied, the styles from the ID selector will take precedence.

If two selectors have the same specificity, the one that appears last in the CSS file will be applied.

To see the effects of specificity, you can modify the styles in the < code>style.css file by commenting or uncommenting specific rules.

CSS Specificity
Selector Type Specificity Value
Inline styles 1000
ID selectors 100
Class selectors 010
Type selectors 001

How to Calculate CSS Specificity

To calculate the specificity of a CSS selector, you can use the following formula:

Specificity = (Inline styles) + (ID selectors * 100) + (Class selectors * 10) + (Type selectors * 1)

For example, if a selector has 1 inline style, 2 ID selectors, 3 class selectors, and 1 type selector, the specificity would be:

Specificity = 1000 + (2 * 100) + (3 * 10) + (1 * 1) = 1000 + 200 + 30 + 1 = 1231

In this case, the specificity value is 1231.

Property Inheritance

In CSS, some properties are inherited by default, while others are not. Inherited properties are passed down from parent elements to child elements.

For example, properties like color, font-family, and line-height are typically inherited, meaning that if you set them on a parent element, child elements will automatically receive those styles unless overridden.

On the other hand, properties like margin, padding, and border are not inherited by default. If you want a child element to inherit these properties, you need to explicitly set them on the child element.

  • Inherited Properties:
    • color
    • font-family
    • line-height
    • font-size
    • font-weight
    • text-align
    • text-decoration
    • text-transform
    • white-space
    • visibility
    • list-style
    • quotes
    • cursor
    • direction
    • font-style
    • text-indent
    • text-shadow
  • Non-Inherited Properties:
    • margin
    • padding
    • border
    • background
    • width
    • height
    • display

CSS Units

CSS Units Types

CSS units are used to define the size of elements in CSS. They can be absolute or relative.

  • Absolute Units: These units are fixed and do not change their computed value regardless of the screen size or resolution
    • px (pixels)
    • pt (points)
    • cm (centimeters)
    • mm (millimeters)
    • in (inches)
  • Relative Units: These units are relative to something else (e.g., the parent element's font size, the root element's font size, or the viewport size). They are crucial for creating responsive and scalable designs.
    • em (relative to the font-size of the element)
    • rem (relative to the font-size of the root element)
    • % (relative to the parent element)
    • vw (viewport width)
    • vh (viewport height)
    • ch (width of the "0" character in the element's font)

px (pixels)

Pixels are the most common unit for measuring length in CSS. They are absolute units and are not affected by the user's font size settings.

Example: This text is 22 pixels in size.

Note: Using pixels can lead to less flexible layouts, especially on different screen sizes.

cm (centimeters)

Centimeters are a unit of measurement commonly used in print media. They are absolute units and are not affected by the user's font size settings.

Example: This text is 1 centimeters in size.

Note: Using centimeters can lead to less flexible layouts, especially on different screen sizes.

em (relative to the font-size of the element)

Em is a relative unit that is based on the font size of the element itself. 1em is equal to the current font size of the element.

Example: This text is 2 times the size of its parent element's font size.

Note: Using em can create flexible designs that adapt to different font sizes, but it may lead to unexpected results if not used carefully.

rem (relative to the font-size of the root element)

Rem is a relative unit that is based on the font size of the root element (usually the <html> element). 1rem is equal to the font size of the root element.

Example: This text is 2 times the size of the root element's font size.

Note: Using rem can create consistent designs across different elements, as it is based on a single reference point.

% (percentage, relative to the parent element's font size)

Percentage is a relative unit that is based on the font size of the parent element. 100% is equal to the current font size of the element's parent.

Example: This text is 150% of its parent element's font size.

Note: Using percentage can create flexible designs that adapt to different font sizes, but it may lead to unexpected results if not used carefully.

vw (viewport width)

Viewport width is a relative unit that represents a percentage of the viewport's width. 1vw is equal to 1% of the viewport's width.

Example: This text is 2% of the viewport's width.

Note: Using viewport units can create responsive designs that adapt to different screen sizes.

vh (viewport height)

Viewport height is a relative unit that represents a percentage of the viewport's height. 1vh is equal to 1% of the viewport's height.

Example: This text is 2% of the viewport's height.

Note: Using viewport units can create responsive designs that adapt to different screen sizes.

ch (character)

Ch is a unit of measurement that is relative to the width of the "0" (zero) character in the element's font.

Example: This text is 5 characters wide.

Note: Using ch can be useful for creating layouts that are based on the width of characters, but it may not be as flexible as other units.
