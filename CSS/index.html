CSS Specificity

CSS Specificity - Theory

CSS specificity determines which styles are applied to an element when there are conflicting styles.

It is calculated based on the types of selectors used in a rule:

  • Inline styles have the highest specificity.
  • ID selectors have higher specificity than class selectors.
  • Class selectors have higher specificity than type selectors.

For example, if an element has both a class and an ID selector applied, the styles from the ID selector will take precedence.

If two selectors have the same specificity, the one that appears last in the CSS file will be applied.

To see the effects of specificity, you can modify the styles in the < code>style.css file by commenting or uncommenting specific rules.

Selector Type Specificity Value

How to Calculate CSS Specificity

To calculate the specificity of a CSS selector, you can use the following formula:

Specificity = (Inline styles) + (ID selectors * 100) + (Class selectors * 10) + (Type selectors * 1)

For example, if a selector has 1 inline style, 2 ID selectors, 3 class selectors, and 1 type selector, the specificity would be:

Specificity = 1000 + (2 * 100) + (3 * 10) + (1 * 1) = 1000 + 200 + 30 + 1 = 1231

In this case, the specificity value is 1231.

Property Inheritance

In CSS, some properties are inherited by default, while others are not. Inherited properties are passed down from parent elements to child elements.

For example, properties like color, font-family, and line-height are typically inherited, meaning that if you set them on a parent element, child elements will automatically receive those styles unless overridden.

On the other hand, properties like margin, padding, and border are not inherited by default. If you want a child element to inherit these properties, you need to explicitly set them on the child element.

  • Inherited Properties:
    • color
    • font-family
    • line-height
    • font-size
    • font-weight
    • text-align
    • text-decoration
    • text-transform
    • white-space
    • visibility
    • list-style
    • quotes
    • cursor
    • direction
    • font-style
    • text-indent
    • text-shadow
  • Non-Inherited Properties:
    • margin
    • padding
    • border
    • background
    • width
    • height
    • display

CSS Units

CSS Units Types

CSS units are used to define the size of elements in CSS. They can be absolute or relative.

  • Absolute Units: These units are fixed and do not change their computed value regardless of the screen size or resolution
    • px (pixels)
    • pt (points)
    • cm (centimeters)
    • mm (millimeters)
    • in (inches)
  • Relative Units: These units are relative to something else (e.g., the parent element's font size, the root element's font size, or the viewport size). They are crucial for creating responsive and scalable designs.
    • em (relative to the font-size of the element)
    • rem (relative to the font-size of the root element)
    • % (relative to the parent element)
    • vw (viewport width)
    • vh (viewport height)
    • ch (width of the "0" character in the element's font)

px (pixels)

Pixels are the most common unit for measuring length in CSS. They are absolute units and are not affected by the user's font size settings.

Example: This text is 22 pixels in size.

Note: Using pixels can lead to less flexible layouts, especially on different screen sizes.

cm (centimeters)

Centimeters are a unit of measurement commonly used in print media. They are absolute units and are not affected by the user's font size settings.

Example: This text is 1 centimeters in size.

Note: Using centimeters can lead to less flexible layouts, especially on different screen sizes.

em (relative to the font-size of the element)

Em is a relative unit that is based on the font size of the element itself. 1em is equal to the current font size of the element.

Example: This text is 2 times the size of its parent element's font size.

Note: Using em can create flexible designs that adapt to different font sizes, but it may lead to unexpected results if not used carefully.

rem (relative to the font-size of the root element)

Rem is a relative unit that is based on the font size of the root element (usually the <html> element). 1rem is equal to the font size of the root element.

Example: This text is 2 times the size of the root element's font size.

Note: Using rem can create consistent designs across different elements, as it is based on a single reference point.

% (percentage, relative to the parent element's font size)

Percentage is a relative unit that is based on the font size of the parent element. 100% is equal to the current font size of the element's parent.

Example: This text is 150% of its parent element's font size.

Note: Using percentage can create flexible designs that adapt to different font sizes, but it may lead to unexpected results if not used carefully.

vw (viewport width)

Viewport width is a relative unit that represents a percentage of the viewport's width. 1vw is equal to 1% of the viewport's width.

Example: This text is 2% of the viewport's width.

Note: Using viewport units can create responsive designs that adapt to different screen sizes.

vh (viewport height)

Viewport height is a relative unit that represents a percentage of the viewport's height. 1vh is equal to 1% of the viewport's height.

Example: This text is 2% of the viewport's height.

Note: Using viewport units can create responsive designs that adapt to different screen sizes.

ch (character)

Ch is a unit of measurement that is relative to the width of the "0" (zero) character in the element's font.

Example: This text is 5 characters wide.

Note: Using ch can be useful for creating layouts that are based on the width of characters, but it may not be as flexible as other units.

Typography

Typography in CSS refers to the style and appearance of text on a web page. It includes font selection, size, weight, line height, letter spacing, and more.

Key properties for typography include:

  • font-family: Specifies the font to be used.
  • font-size: Sets the size of the font.
  • font-weight: Defines the thickness of the font.
  • line-height: Controls the space between lines of text.
  • letter-spacing: Adjusts the space between characters.
  • text-align: Sets the alignment of text (left, right, center, justify).
  • text-decoration: Adds decoration to text (underline, overline, line-through).
  • text-transform: Controls capitalization (uppercase, lowercase, capitalize).

Display

The display property in CSS is used to control the layout of an element. It determines how an element is displayed on the page and how it interacts with other elements.

Key values for the display property include:

  • block: The element is displayed as a block-level element, taking up the full width available and starting on a new line.
  • inline: The element is displayed as an inline element, meaning it does not start on a new line and only takes up as much width as necessary.
  • inline-block: The element is displayed as an inline-level block container, allowing it to have width and height while still flowing with text.
  • flex: The element is displayed as a flex container, allowing for flexible layouts and alignment of child elements.
  • grid: The element is displayed as a grid container, allowing for a two-dimensional layout of child elements in rows and columns.
  • none: The element is not displayed at all, and it does not take up any space in the layout.
  • table: The element is displayed as a table, allowing it to behave like a table element in HTML.
  • table-row: The element is displayed as a table row, allowing it to behave like a row in a table.
  • table-cell: The element is displayed as a table cell, allowing it to behave like a cell in a table.
  • list-item: The element is displayed as a list item, allowing it to behave like an item in a list.

  • Block elements can contain other block elements and inline elements.

  • margin-top, margin-bottom: We can set margin left, right, top, and bottom. The top and bottom margins will push surrounding elements away.

  • float: Applying float to a block element will change its computed display value to inline-block, allowing it to behave like an inline element in terms of layout, but it will still take up the full width available.

display: inline

  • Inline elements do not start on a new line.
  • Inline elements can be nested within block-level elements.
  • width, height: We can not give inline elements a specific width and height. These properties are ignored. An inline element's width and height are determined solely by its content.
  • margin-top, margin-bottom: We can set margin left and right, but not top and bottom.

Some common inline elements:

  • <a>
  • <strong>
  • <em>

display: inline-block

inline-block elements are similar to inline elements, but they can have width and height set. They also respect top and bottom margins and padding.

  • Inline-block elements do not start on a new line.
  • Inline-block elements can have width and height set.
  • margin: We can set margin for all sides (top, right, bottom, left).
  • padding: We can set padding for all sides (top, right, bottom, left).
  • float: Applying float to an inline-block element will change its computed display value to block, making it behave like a block-level element in terms of layout.

Some common inline-block elements:

  • <img>
  • <input>
  • <button>

display: none

Elements with display: none are not rendered at all (they have no effect on layout); they are completely removed from the document flow.

Example: This text will not be displayed.

  • Elements with display: none do not take up any space in the layout.
  • They are not visible to users and do not affect the layout of other elements.
  • They can be toggled using JavaScript or CSS transitions to show/hide elements dynamically.

display: flex

Flexbox is a layout model that allows elements to be aligned and distributed within a container. It provides a more efficient way to lay out, align, and distribute space among items in a container, even when their size is unknown or dynamic.

  • Flex containers can be created by setting the display property to flex or inline-flex.
  • Flex items are the direct children of a flex container and can be manipulated using various flex properties.
  • flex-direction: Defines the direction in which flex items are placed in the flex container (row, column, row-reverse, column-reverse).
  • justify-content: Aligns flex items along the main axis (start, end, center, space-between, space-around).
  • align-items: Aligns flex items along the cross axis (stretch, center, start, end, baseline).
  • flex-wrap: Controls whether flex items should wrap onto multiple lines (nowrap, wrap, wrap-reverse).

Some common flex properties:

  • flex-grow: Defines the ability for a flex item to grow if necessary.
  • flex-shrink: Defines the ability for a flex item to shrink if necessary.
  • flex-basis: Defines the default size of a flex item before space is distributed.

display: grid

CSS Grid Layout is a two-dimensional layout system that allows you to create complex layouts using rows and columns. It provides a powerful way to design responsive and flexible layouts.

  • Grid containers can be created by setting the display property to grid or inline-grid.
  • Grid items are the direct children of a grid container and can be manipulated using various grid properties.
  • grid-template-columns: Defines the number and size of columns in the grid.
  • grid-template-rows: Defines the number and size of rows in the grid.
  • grid-gap: Sets the gap between grid items (both row and column gaps).
  • grid-auto-flow: Controls how auto-placed items are placed in the grid (row, column, dense).

Some common grid properties:

  • grid-column: Specifies how many columns an item should span.
  • grid-row: Specifies how many rows an item should span.
  • grid-area: Defines a named area within the grid for placing items.

CSS Positioning

CSS positioning allows you to control the placement of elements on a web page. There are several positioning schemes in CSS:

  • static: The default positioning scheme. Elements are positioned according to the normal document flow. They are not affected by top, right, bottom, or left properties.
  • relative: Elements are positioned relative to their normal position in the document flow. The top, right, bottom, and left properties offset the element from its original position.
  • absolute: Elements are positioned relative to the nearest positioned ancestor (an ancestor with a position other than static). If there is no such ancestor, the element is positioned relative to the initial containing block (usually the viewport).
  • fixed: Elements are positioned relative to the viewport. They remain in the same position even when the page is scrolled. The top, right, bottom, and left properties offset the element from the viewport edges.
  • sticky: Elements are positioned based on the user's scroll position. They behave like relative elements until a specified scroll position is reached, at which point they become fixed.
